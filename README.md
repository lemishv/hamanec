# Гаманець

Облік витрат і накопичень у власних журналах. Single-file PWA для Android.

## Що це

- **Журнали витрат** — побутові, проєктні, на поїздку. З опційним бюджетом, темпом і прогнозом.
- **Банки накопичень** — з ціллю і опційною датою досягнення.
- **Категорії** — гнучкі, для кожного журналу свої.
- **Статистика** — крос-журнальна, по валютах, з опційним перерахунком у базову валюту за заданими курсами.
- **Без серверів.** Дані зберігаються у `localStorage` браузера. Нічого не передається назовні.
- **Працює офлайн.** Після першого відкриття все функціонує без мережі.
- **Експорт CSV** з кожного журналу.

## Файли в репо

```
index.html              ← сам додаток (один файл, ~100 КБ)
manifest.json           ← маніфест PWA
sw.js                   ← Service Worker (офлайн + кеш)
icon-192.png            ← іконки PWA
icon-512.png
icon-maskable-192.png   ← maskable для Android adaptive icons
icon-maskable-512.png
apple-touch-icon.png    ← iOS (на випадок якщо відкриєш на iPhone)
favicon-32.png          ← для вкладок браузера
```

## Деплой на GitHub Pages

1. Створи репозиторій (наприклад `hamanets`) на GitHub.
2. Завантаж усі файли з цього архіву в корінь репо.
   - **Важливо:** мобільний браузер при завантаженні часто додає суфікс (`index-1.html`). Якщо так — після завантаження через GitHub web UI натисни на файл, потім олівець «Edit» → перейменуй на чистий `index.html`. Те саме для `sw.js`.
3. Settings → Pages → Source: `main` / `(root)` → Save.
4. Через ~1 хвилину додаток буде доступний за адресою `https://<твій-логін>.github.io/hamanets/`.

## Встановлення на головний екран Android

1. Відкрий адресу в Chrome на телефоні.
2. Меню (три крапки) → **«Встановити застосунок»** або **«Додати на головний екран»**.
3. Іконка «Г» з'явиться на робочому столі. Відкривається у власному вікні без браузерної шапки.

## Як оновлювати

1. Заміни `index.html` у репо (через GitHub web UI: натисни на файл → олівець → встав новий код).
2. Якщо змінив `sw.js`, **обов'язково** збільш версію кешу в першому рядку:
   ```js
   const CACHE = 'hamanets-v2';
   ```
   Це змусить Service Worker почистити старий кеш і завантажити нові файли.
3. Якщо змінив тільки `index.html` — версію кешу теж краще збільшити, бо інакше старий може якийсь час залишитись у клієнтів.
4. Закрий і знову відкрий додаток на телефоні — побачиш нову версію.

## Кешування — перший підозрюваний

Якщо здається що зміни не зайшли:
- Перевір що GitHub Pages показує новий файл (відкрий у браузері, переглянь джерело).
- У встановленому PWA: можна тимчасово відкрити в звичайному Chrome (не з іконки) і зробити hard refresh.
- При сумнівах — збільшити версію `CACHE` у `sw.js`.

## Технічно

- **HTML/CSS/JS у одному файлі** — ніяких зборок, тулчейнів, npm.
- **Service Worker** кешує статичні ресурси. Стратегія: network-first для HTML (щоб бачити оновлення), cache-first для іконок/маніфесту.
- **Шрифти** (Fraunces, JetBrains Mono) — з Google Fonts. Кешуються браузером після першого завантаження. Якщо інтернету немає — підвантажиться системний serif/mono fallback.
- **Зберігання** — `localStorage`, ключ `hamanets_v1`. Об'єм невеликий (5-10 МБ ліміт браузера — тисячі транзакцій влізуть).
